# PDF Checkmaker

Консольный Python-скрипт для генерации PDF по данным из `CSV` или `JSON` и HTML-шаблонам.

## Что умеет

- читает все `CSV` и `JSON` файлы из папки `data`
- читает все HTML-шаблоны из папки `templates`
- показывает нумерованное меню выбора файлов
- показывает список доступных чеков по `invoice_id`
- подставляет данные в HTML-шаблон
- генерирует PDF через `WeasyPrint`
- сохраняет результат в `output`
- автоматически открывает PDF в системной программе

## Структура проекта

```text
PDF maker/
  main.py
  README.md
  requirements.txt
  data/
    sample_invoices.json
  templates/
    invoice.html
  output/
```

## Требования

- Python 3.10+
- зависимости из `requirements.txt`

## Установка

Из корня проекта:

```bash
pip install -r requirements.txt
```

## Запуск

Из корня проекта:

```bash
python main.py
```

## Первый запуск

В проект уже добавлены готовые примеры:

- `data/sample_invoices.json`
- `templates/invoice.html`

Поэтому после установки зависимостей можно сразу запускать `main.py` без дополнительной подготовки.

После запуска скрипт:

1. покажет доступные файлы данных
2. покажет доступные HTML-шаблоны
3. попросит выбрать файл данных
4. попросит выбрать шаблон
5. покажет список доступных `invoice_id`
6. создаст PDF и откроет его автоматически

## Поддерживаемые форматы данных

### JSON

Поддерживаются такие варианты:

- массив объектов
- объект с ключом `invoices`
- один объект счета

Пример:

```json
[
  {
    "invoice_id": "INV-1001",
    "date": "2026-04-10",
    "customer_name": "ООО Ромашка",
    "total": "12500.00",
    "items": [
      {
        "name": "Услуга",
        "quantity": 1,
        "price": 12500,
        "amount": 12500
      }
    ]
  }
]
```

### CSV

CSV должен содержать колонку с идентификатором счета. Подходят названия:

- `invoice_id`
- `invoice id`
- `invoiceid`
- `invoice`
- `id`

Строки с одинаковым `invoice_id` автоматически объединяются в один чек, а строки попадают в `items`.

Пример:

```csv
invoice_id,date,customer_name,name,quantity,price,amount,total,status
INV-2001,2026-04-10,ООО Ромашка,Услуга 1,1,5000,5000,8000,Оплачен
INV-2001,2026-04-10,ООО Ромашка,Услуга 2,1,3000,3000,8000,Оплачен
```

## Шаблоны

Шаблоны лежат в папке `templates` и используют плейсхолдеры вида:

```html
<h1>Счет {{invoice_id}}</h1>
<p>Клиент: {{customer_name}}</p>
{{items_table}}
```

Доступные полезные поля:

- `{{invoice_id}}`
- `{{generated_at}}`
- `{{source_file}}`
- любые простые поля из JSON или CSV, например `{{date}}`, `{{customer_name}}`, `{{total}}`
- `{{items_table}}` для автоматической таблицы позиций
- `{{invoice_json}}` для полного JSON счета
- `{{items_json}}` для JSON списка позиций

## Шрифты и кириллица

Скрипт пытается подключить:

- `DejaVu Sans`
- `Roboto`

Если файлы шрифтов доступны в системе, они будут использованы для PDF. Это нужно для корректного отображения кириллицы на Windows и macOS.

## Выходной результат

Сгенерированные файлы сохраняются в папку `output` в формате:

```text
invoice_<invoice_id>.pdf
```

## Возможные проблемы

Если `weasyprint` не установлен, импорт завершится ошибкой. Решение:

```bash
pip install -r requirements.txt
```

Если PDF создался, но кириллица отображается некорректно, проверь, установлен ли в системе `DejaVu Sans` или `Roboto`.
